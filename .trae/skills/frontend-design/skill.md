# 前端设计（生产级 UI，明确审美方向）

## 本技能做什么

交付真实可运行的前端 UI 实现，并保持强烈且一致的视觉观点：
- 坚持一个清晰的审美方向（不要“什么都来一点”）
- 输出生产级代码（不是效果图或伪代码）
- 建立统一的字体体系、配色体系、间距尺度与交互状态
- 更偏好少量高质量的“高光时刻”，而不是零碎的微交互堆砌
- 避免通用“AI 审美默认值”
- 尽量使模块整体看起来充满整个屏幕，不要让模块看起来“局促”
## 兼容性

- 适用技术栈：React / Vue / HTML+CSS（优先沿用仓库已有栈）
- 样式方案：CSS Modules / styled-components / Tailwind / 原生 CSS（与项目保持一致）
- 动效：默认使用 CSS 动画；仅当项目已引入动效库时再使用
- 可访问性：键盘焦点态、可读对比度、语义化结构

## 工作流程

必须完整走完以下闭环（不要做到一半就停）：

1. **理解意图**
   - 明确产物：页面 / 组件 / 看板 / 布局 / 主题
   - 记录约束：框架、路由、组件库、性能、无障碍
2. **选择一个大胆方向**
   - 给方向命名（如“编辑部极简”“工业工具风”“复古未来主义”）
   - 定义配色 + 字体搭配 + 间距节奏
3. **实现可运行代码**
   - 复用项目既有模式（状态管理、数据请求、路由等）
   - 构建响应式布局与交互态（hover/focus/active/disabled）
4. **精修**
   - 收紧层级与间距
   - 用“有意图的细节”替换通用默认值（边框、阴影、纹理等）
5. **验证**
   - 有条件时运行 lint/build/typecheck
   - 冒烟测试：空态/错误态/加载态

## 设计思考（承诺规则）

在编码之前先理解上下文，并承诺一个明确的审美方向：
- **目的**：这个界面解决什么问题？谁会使用？
- **调性**：选择一个“极端值”作为方向：极简、极繁、复古未来、自然有机、精致奢华、玩具感、编辑部风、粗野主义、装饰艺术、柔和粉彩、工业工具风等。用它们启发，但最终必须收敛到一个真实可执行的方向。
- **约束**：技术约束（框架、性能、无障碍等）。
- **差异化**：什么会让人“一眼记住”？用户会记住的那个点是什么？

**关键要求**：选择一个清晰的概念方向，并用精准执行落地。极繁或极简都可以，关键是“有意图”而不是“强度大”。

然后实现可运行的代码（HTML/CSS/JS、React、Vue 等），要求：
- 生产级、可用
- 视觉上有冲击、可记忆
- 有一致且明确的视觉观点
- 细节打磨到位

## 前端审美指南

重点关注：
- **字体**：选择漂亮、独特、有性格的字体。避免 Arial/Inter 等过度常见字体；用更有记忆点的展示字体 + 更克制的正文字体做搭配。
- **配色与主题**：坚持一个统一的审美体系。使用 CSS 变量保证一致性。明确的主色 + 锐利点缀通常优于“平均分配”的胆小配色。
- **动效**：用动画做关键效果与微交互。HTML 优先 CSS-only；React 仅在项目已有动效库时使用。优先打造少数高光时刻（例如一次有节奏的入场动画），不要散落一堆小动效。用滚动触发与 hover 状态做惊喜点。
- **空间构图**：非对称、叠层、对角线动势、打破网格的元素、充足留白或受控密度。让所有元素的排列组合在横向上能充满整个屏幕，不能在横向上有留白
            case: 第一层有一个元素，第二层有两个元素，那么第一层的那一个元素应该充满屏幕宽度，第二层的两个元素应该各占屏幕宽度的一半（包括padding）
- **背景与细节**：营造氛围与层次，不要只用纯色背景。按整体风格加入纹理与效果：渐变网格、噪点纹理、几何图案、叠加透明层、戏剧性阴影、装饰边框、自定义光标、颗粒叠层等。

严禁使用通用 AI 风格：过度常见的字体族（Inter/Roboto/Arial/system fonts）、陈词滥调的配色（尤其是白底紫色渐变）、可预测的布局与组件套路、缺乏情境与性格的模板化设计。

要有创造性地解读需求，做出“像真的为这个场景设计过”的意外选择。每次生成都不应雷同：可以在亮/暗主题、字体、风格方向上做变化。不要跨多次生成反复收敛到同一种常见选择（例如总用 Space Grotesk）。

**重要**：实现复杂度要匹配审美方向。极繁需要更丰富的层次、动效与细节；极简/精致需要克制、精准，对间距、字体、微妙细节要求更高。优雅来自“执行到位”，不是堆料。

记住：你有能力做出非常强的创意与落地。不要保守——真正跳出模板、坚定执行一个方向，才能做出“产品级且有记忆点”的界面。

## 输出要求

该技能触发后必须输出：

- 1–3 行的“设计方向”摘要（用于锚定后续决策）
- 可运行的代码（不是伪代码），并遵循项目约定与风格
- 最小集成说明：改了哪些文件 / 如何预览

## 测试提示（2–3）

用这些真实用户式输入来检查触发与输出质量：

1. “把这个页面整体美化一下，要高级、克制、有品牌感。”
2. “做一个数据看板页面，包含几个卡片指标和一个列表，要求很有设计感。”
3. “帮我写一个 React 组件：设置面板 + 表单 + 保存按钮，要看起来像产品级。”
4. "重构一下这个页面的UI"
